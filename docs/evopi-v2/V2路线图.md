## P1 · 接下来最该做

### 1. 主动缓存编排（模块 2 从"测"到"提"）
- **现状**：V1 只**观测**缓存命中率（`computeCacheHitRate` / `cost.cache`）。
- **V2 增量**：主动编排 prompt 布局——稳定系统提示 → 稳定工具定义 → 易变任务上下文，最大化 Anthropic 自动缓存命中。
- **验收设想**：同一多轮任务，编排前 vs 后的命中率对比（用 V1 已有的命中率口径量化）——**天然是个可度量的 demo**。
- **类型**：纯加法。依赖 Cost ✅。

### 2. Codebase RAG（代码库语义检索）
- **现状**：V1 待办清单明确"cost trace 可见前不做 RAG"——现在解锁。
- **V2 增量**：给 agent 语义检索代码库的能力，减少读错/读多余文件的 token 浪费。
- **要想清楚**：检索引擎选型（可复用 mempalace 那套**本地优先、零 API** 的关键词加权思路，省钱可控，见[调研](../Pi开源参考项目调研_2026-06-30.md)）。
- **类型**：纯加法。依赖 Cost 可观测 ✅。

## P2 · 更远（含需要 ADR 的架构改造）

### 3. 自进化 Skill Factory（模块 3+6，借鉴 darwin-skill）
- **V2 增量**：从失败 trace 自动生成候选 skill → 多评估器打分 → 保留改进/回滚。V1 已做半自动棘轮（人 `gate`），V2 让它更自动。
- **⚠ 架构点**：触碰 V1"自动决策留 V2"约定 → **先看 [ADR-0001](架构决策记录/ADR-0001-v2是否解冻自动决策约定.md)**（建议分级解冻：自动提议、人采纳）。
- **类型**：含架构改造。依赖 Skill/Eval ✅。

### 4. 执行自动 rewind（模块 4）
- **V2 增量**：检测失败（测试挂/高危未解决）自动回退到最近安全 checkpoint。V1 只有手动 rewind。
- **⚠ 架构点**：自动回退是自动决策 → 同样受 [ADR-0001](架构决策记录/ADR-0001-v2是否解冻自动决策约定.md) 约束。
- **类型**：含架构改造。依赖 checkpoint 机制 ✅。

### 5. MCP broker + 远程沙箱后端（模块 5）
- **V2 增量**：接入 MCP（标准化外部工具）+ 把高危操作放沙箱隔离执行——高危从"拦下"升级到"隔离里安全跑"。
- **要想清楚**：V1 待办清单要求"policy 结构稳定后再加 MCP"——先确认 policy schema 冻结。
- **类型**：纯加法（但工作量大）。

### 6. 工具级超时（模块 5，V1 遗留）
- **背景**：V1 因 Pi 扩展 API 不暴露单工具 AbortController 而改 V2（见 V1 变更日志）。
- **V2 增量**：若 Pi 后续开放相关 API，补单工具超时；否则维持 bash 原生超时兜底。
- **类型**：纯加法。依赖 Pi API。

## 怎么用这张表

1. **选一条**（通常从 P1 起，或你当下最有感觉的）。
2. 若标了 **⚠ 架构点** → 先把对应 ADR 的"决定"拍板（改状态为已接受/已否决）。
3. 进实现 → 走 V1 的 [impl-harness 流水线](../evopi-v1/impl/README.md)：冻结契约 → 拆文件 → 核心逻辑 → 自测（新代码补测试进 `tests/`）→ 小步中文提交。
4. 完成后回这张表打勾、更新状态。

> 新想法随时进[灵感池](灵感池.md)，够格了再升级到这张表——路线图跟着演化，不是刻死的。
