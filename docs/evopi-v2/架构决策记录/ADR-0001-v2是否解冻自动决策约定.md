# ADR-0001：V2 是否解冻「自动决策留 V2」这条 V1 冻结约定

- **状态**：提议中（Proposed）
- **日期**：2026-07-02
- **相关**：[灵感池](../灵感池.md) 的「技能自进化 Skill Factory」「执行自动 rewind」两条；影响模块 3/4/6；触碰 V1 冻结约定第 5 条「保守默认——自动决策留 V2」。

## 背景（Context）

V1 定了一条全局冻结约定（见 [impl/进度.md](../../evopi-v1/impl/进度.md) 冻结约定第 5 条）：

> **保守默认**：总是拦高危、无 UI fail-safe、**评分不猜、自动决策留 V2**。

于是 V1 里所有"要不要接受一个进化/要不要回退"都是**人拍板**：技能棘轮是半自动（`gate enable|reject` 由人裁决）、执行 rewind 是手动。

V2 的两个高价值方向都想突破这条：
- **自进化 Skill Factory**（借鉴 darwin-skill）：从失败 trace 自动生成候选 → 多评估器打分 → **自动**保留改进/回滚。
- **执行自动 rewind**：检测失败时**自动**回退到 checkpoint。

问题：这两个都属于"agent 自动做治理决策"，正是 V1 刻意推迟的。**V2 要不要解冻？解冻到什么程度？**

## 备选方案（Options）

1. **完全解冻，全自动**：候选打分过阈值就自动 enable、失败就自动 rewind，无人参与。
   - 优点：最"智能"，演示效果炫。
   - 代价 / 爆炸半径：**大**。自动决策一旦误判（评分被 reward-hack、rewind 回退到更糟的点），会在无人值守下累积错误，且与 V1"安全>便利""fail-safe"的整体气质冲突。难测、难解释。

2. **分级解冻（推荐方向）**：区分"决策"和"执行"——
   - 允许**自动"提议"**（自动生成候选、自动算出"该回退到哪"、自动跑评估给出建议）；
   - 但**保留人在关键闸口拍板**（enable / 真正 rewind 仍需一次确认），或只在"低风险 + 高置信"时才自动、其余留人。
   - 优点：拿到自动化的大部分收益（省掉人工跑评估/找 checkpoint 的苦力活），又不放弃"最终由人负责"的安全底线。可增量放开。
   - 代价：不是"纯全自动"那么炫，但更可信、可解释、可测。

3. **不解冻，维持 V1 半自动**：
   - 优点：零风险，V1 气质一致。
   - 代价：V2 少了最有"进化感"的卖点；darwin-skill 那套棘轮的价值发挥不出来。

## 决定（Decision）

> ⏳ **待定** —— 这是示范 ADR，决定留给你（项目作者）拍板。
>
> 我的**建议倾向方案 2（分级解冻）**：V2 把"自动提议"放开，"自动执行关键治理动作"仍需一次人确认或严格限定在低风险高置信区间。理由见下。

## 理由（Rationale）

- EvoPi 的**核心气质是"可信治理"**，不是"最聪明的 agent"。全自动决策（方案 1）恰恰削弱这个气质——一个会自己误判又无人兜底的治理层，比不治理更危险。
- 方案 2 拿到了自动化 80% 的收益（自动生成候选、自动评估、自动定位回退点——这些都是纯加法、可测、可回滚），只在最后 20%（"是否采纳"）保留人的闸口。**收益/风险比最优**。
- 它也**兼容 V1 的冻结约定**：不是"推翻"，而是"精细化"——把"自动决策留 V2"改成"自动**提议**在 V2 放开，自动**采纳**仍受控"。这是一次受控的解冻，不是拆掉安全网。

## 影响（Consequences）

- **正面**：V2 能落地自进化/自动 rewind 的大部分价值，且保持可信、可解释、可测、可增量。
- **代价 / 风险**：需要在实现时明确划出"自动提议"与"自动采纳"的边界，并为自动部分补测试（守住"提议不等于生效"）。
- **爆炸半径**：触碰 V1 冻结约定第 5 条 → 若采纳方案 2，需在 [impl/进度.md 变更日志](../../evopi-v1/impl/进度.md#变更日志改冻结约定必须记这里含理由) 记一笔："自动决策留 V2" 精细化为 "自动提议放开 / 自动采纳受控"，理由链接本 ADR。涉及模块 3（Skill Factory）、4（自动 rewind）、6（评估器）。
- **后续动作**：
  - 决定后更新本 ADR 状态为 `已接受/已否决`；
  - 采纳方案 2 → 在 [V2路线图](../V2路线图.md) 给"自进化棘轮""自动 rewind"两条标注"按 ADR-0001 分级解冻"；
  - 实现时给"自动提议→人采纳"的边界加集成测试。
