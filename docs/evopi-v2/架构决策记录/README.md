# 架构决策记录（ADR）

> ADR = Architecture Decision Record。**一个重要决策一篇**，记录：背景、备选、决定、理由、影响。
>
> 什么时候写：定性为 **B 架构优化** 或 **C 推翻旧决策** 的输入（见[演化流程](../演化流程.md)第③步）。纯加法的新功能不用写 ADR，进[路线图](../V2路线图.md)即可。

## 为什么写 ADR

- **给未来的自己**：三个月后你会忘"当初为什么这么改"。ADR 是你和过去的自己对话的方式。
- **给面试官**：翻到 ADR，看到的是"这个人会权衡、会演化系统、改动前会评估影响"——比"会写代码"值钱得多。
- **约束冲动**：写 ADR 逼你把"我想改"变成"我为什么改、代价是什么、有没有更小的做法"。

## 命名与状态

- 文件名：`ADR-000X-<短标题>.md`（四位序号，从 0001 起）。
- 状态：`提议中 (Proposed)` → `已接受 (Accepted)` / `已否决 (Rejected)` / `已被取代 (Superseded by ADR-000Y)`。

见 [ADR-0000-模板.md](ADR-0000-模板.md)。写新 ADR 时复制它。
