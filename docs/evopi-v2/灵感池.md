## 池子

> 下面几条是**种子**——从 V1 的[模块待办清单](../evopi-v1/模块待办清单.md)和[开源调研](../Pi开源参考项目调研_2026-06-30.md)里已经埋着的"下一步"提出来的，示范这个池子怎么用。新想法往下加。

### 2026-07-02 · Codebase RAG（代码库检索增强）
- **来源**：V1 模块待办清单里 Cost Harness 明确写了"Do not implement codebase RAG until cost traces are visible"——现在 cost trace 已可见，解锁条件达成。
- **它主张什么**：给 agent 一个能按语义检索代码库的能力，减少"读错文件/读多余文件"的 token 浪费。
- **对 EvoPi 的可借鉴点**：模块 2（Cost）的自然延伸——先有成本可观测，再用检索降成本。
- **定性**：🔬已提炼 → A 新功能
- **去向**：→ V2路线图（P1，依赖 Cost 已完成 ✅）

### 2026-07-02 · 主动缓存策略（cache-friendly prompt 主动编排）
- **来源**：V1 模块待办清单 Cost Harness："stable system prompt → stable tools → volatile task context"，V1 只做了观测，没做主动编排。
- **它主张什么**：主动把稳定内容前置、易变内容后置，最大化 Anthropic 自动缓存命中率。
- **对 EvoPi 的可借鉴点**：模块 2 从"测命中率"升级到"主动提高命中率"。
- **定性**：🔬已提炼 → A 新功能
- **去向**：→ V2路线图（P1）

### 2026-07-02 · 技能自进化 Skill Factory（棘轮全自动化）
- **来源**：开源调研的 **darwin-skill**（棘轮规则 + 9 维打分 + spawn N 个评估子代理 + git-revert 兜底）。
- **它主张什么**：从失败 trace 自动生成候选 skill → 多评估器打分 → 只保留可测量的改进、其余回滚。
- **对 EvoPi 的可借鉴点**：模块 3+6 联动。V1 已做半自动棘轮（人拍板 `gate`），V2 让它更自动。
- **定性**：🔬已提炼 → A 新功能（**注意含 B 架构**：全自动裁决触碰 V1"评分不猜/自动决策留 V2"的冻结约定 → 需配 ADR）
- **去向**：→ V2路线图（P2）+ 待写 ADR（涉及解冻"自动决策留 V2"）

### 2026-07-02 · MCP broker + 远程沙箱后端
- **来源**：V1 模块待办清单 Tool Runtime："Add MCP broker only after the policy profile shape is stable" + "remote sandbox backends"。
- **它主张什么**：工具运行时接入 MCP（标准化外部工具）+ 把高危操作放进沙箱执行。
- **对 EvoPi 的可借鉴点**：模块 5 的重大扩展；沙箱能让"高危操作"从"拦下"升级到"隔离里安全跑"。
- **定性**：🔬已提炼 → A 新功能
- **去向**：→ V2路线图（P2，依赖 policy 结构稳定）

### 2026-07-02 · 执行自动 rewind（出错自动回退到 checkpoint）
- **来源**：V1 模块待办清单 Execution："Add checkpoint hooks ... before implementing rewind UX"——V1 只做了手动 rewind。
- **它主张什么**：检测到失败（测试挂/高危未解决）时自动回退到最近安全 checkpoint。
- **对 EvoPi 的可借鉴点**：模块 4 从"手动 rewind"到"自动 rewind"。
- **定性**：🔬已提炼 → A 新功能（含 B：自动回退是自动决策，同样触碰"自动决策留 V2"约定）
- **去向**：→ V2路线图（P2）+ 可能需 ADR
