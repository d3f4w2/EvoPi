## 功能 1 — Codebase RAG

**增量设计**：[codebase-rag.md](../增量设计/codebase-rag.md)（现状→增量→前后区别→怎么做→验收）。

### 冻结契约（开工即冻，改动记变更日志）

1. **纯加法**：不改任何已冻结的事件字段 / policy / 现有模块（cost/memory/job/tools/eval）行为。
2. **新事件 `rag.retrieve`**：字段 `{ query, hits: [{path, startLine, endLine, score}], k, indexedFiles }`。**只进 JSONL，不 anchor**（逐次检索是观测，守 Anchor-only）。加字段向后兼容，改名/删字段记变更日志。
3. **注入契约**：`context` 钩子注入一条 `customType="evopi-rag"`、`display:false` 的消息；**先滤掉上轮 evopi-rag 消息再注入**（防叠加，同 memory 模式）；与 `evopi-memory` 注入并存、互不干扰。
4. **零 API**：检索本地词频打分，不调 LLM、不发网络请求（与 memory 检索同哲学）。
5. **注入软上限**：Top-K ≤ 8 片段、总注入字符有上限（防爆 context），空 query / 无命中不注入。
6. **索引范围**：跳过 node_modules / dist / .git / pi / .pi/evopi 等；只索引文本源码。

### 实现清单

- [x] `rag.ts`：`tokenize`/`scoreChunk`/`chunkFile`/`buildIndex`/`retrieveTopK`/`renderInjection`（借鉴 memory.ts 的本地打分，另拆 camelCase、路径命中 +2）。
- [x] `rag.ts`：`registerRag(pi, shared)` —— `context` 钩子注入（先滤旧 evopi-rag 再注入）+ 记 `rag.retrieve` + `/evopi-rag`（状态 / reindex）。
- [x] index.ts：加 `registerRag(pi, shared)`（模块 6 之后）。
- [x] tests/：`rag.test.ts` —— 20 断言覆盖 tokenize/chunkFile/scoreChunk/retrieveTopK/renderInjection/lastUserText 及边界。

### 验收判据（Done = 全绿）

1. ✅ 检索：相关片段排前、无关不进 Top-K；长文件切成带行号片段（rag.test 覆盖）。
2. ✅ 注入：`customType="evopi-rag"`、`display:false`、含文件路径行号、受 6000 字符上限约束；空 query/无命中不注入；context 钩子先滤旧 evopi-rag 再注入（不叠加）。
3. ✅ `npm test` 49 → **69 全绿**（新增 20 条 RAG 断言），无回归。
4. ✅ `rag.retrieve` 只 `recorder.record`（默认只进 JSONL，未传 anchor），守 Anchor-only。
5. ✅ strict tsc 对 11 个 .ts（+rag.ts）0 错。

**验收证据（2026-07-02，全绿）**：真码离线驱动——tests/rag.test.ts 直接 import 真实 rag.ts，69 断言全过；typecheck 0 错。遗留：真 Pi 会话下 context 注入的端到端 run 待有环境补（注入路径已在 memory 生产验证过同款挂点）。
