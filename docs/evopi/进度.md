### 已查清的 Pi 缓存事实（决策依据，源码已确认）

- Pi 在 Anthropic 上**自动**打 `cache_control`，覆盖 ①每个 system text 块 ②最后一个 tool 定义 ③最后一条 user 消息的最后一个块；Anthropic 总共只允许 4 个断点，Pi 已占 3 个。
- 由 `cacheRetention` 控制：`none` / `short`(5m 默认) / `long`(1h，模型支持时)，env 变量 `PI_CACHE_RETENTION=long`。
- `before_provider_request` 的 `event.payload` 是裸 Anthropic 请求体，handler 返回值能真正改写并发给 provider。
- `after_provider_response` 事件**不含 usage**；真实 token（`cacheRead`/`cacheWrite`/`cacheWrite1h`）在 `ctx.sessionManager.getLeafEntry()` 的 assistant message `.usage` 里。
- `getContextUsage().tokens` 是**估算**，compaction 后到下次响应前为 null。

## 已锁定的全局决策（跨模块约束）

### 核心模型

- **Session** = 用户与 agent 的主线语义记录（只记重要语义事件）
- **Session Tree** = Session 的 resume / fork / branch 结构
- **Trace** = agent 执行过程的完整观测证据链（记所有执行细节）
- **Eval** = 对 Trace 的评分
- **Memory / Skill** = 从高分/低分 Trace + Eval 沉淀的长期资产

### Trace 关键决策

- 不全量双写 trace 到 session；也不完全断开 session 与 trace。
- 采用 **Anchor-only session writes**：session 只写 traceId / path / summary / checkpoint 等关键锚点。
- 完整 Trace 进 **JSONL**（SQLite 列为后续规划，尚未实现）。
- Trace 是底座；Cost / Memory / Job / Tools / Eval 是上层消费者。

### 工程约定

- 工作目录 `D:\evopi`（内层 `pi/` 是只读参考仓库，已 gitignore）。
- commit message 用中文，小步提交。
- 规划阶段默认只讨论不写代码（除非明确要求实现）。
- 回复与文档尽量用中文。
- **每个决策都要写明「为什么这么选」的理由**（规则见 [README.md](README.md)）。
