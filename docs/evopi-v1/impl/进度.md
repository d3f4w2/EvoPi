## 冻结的全局约定（实现期不得漂移）

这些一旦开工就**冻结**，改动需显式记一笔理由到本表「变更日志」：

1. **事件词表冻结**：各模块 `*.*` 事件的名字与字段以对应方案文档的 schema 为准（如 Cost 见 [02-上下文成本机制.md](../02-上下文成本机制.md) 决策 4）。下游 SQLite / V2 自动化都依赖它不变。**加字段可以（向后兼容），改名/删字段要记变更日志。**
2. **Anchor-only**：只有「资产产生/审批/关键治理决策」写 session anchor，逐次观测只进 JSONL。判据见 [00-整体架构.md](../00-整体架构.md) 第四节。
3. **安全 > 资源**：`tool_call` 是单 handler，模块 4（安全准入）先于模块 5（预算资源）。
4. **共享 policy**：模块 3/4/5 读同一份 `.pi/evopi/policy.json`，不各造一套。
5. **保守默认**：总是拦高危、无 UI fail-safe、评分不猜、自动决策留 V2。

### 模块 2 — Cost 上下文成本

**契约**：[02-上下文成本机制.md](../02-上下文成本机制.md)（挂点见二节、事件 schema 见决策 4、口径见处理流程）。**冻结**：`cost.request`/`cost.cache`/`cost.pressure` 三事件字段。

**实现清单**：
- [x] 从 `index.ts` 迁出 `/evopi-cost` 与 provider 钩子逻辑到 `cost.ts`，`index.ts` 只 `registerCost(pi, shared)`。（顺带抽 `trace.ts` 底座——cost 依赖它，是模块 2 的②接线步骤）
- [x] `before_provider_request`：记 `cost.request` + 查 `getContextUsage().percent` 跨阈值告警。
- [x] `after_provider_response`：读最近 assistant `.usage` 算命中率，记 `cost.cache`，in-memory 累加（按 entry id 去重，防错误响应重复累加）。
- [x] `cost.pressure`：首次跨 80/90/95% 各记一次（不每轮记；回落重置 floor 允许再报）。
- [x] `/evopi-cost`（摘要）+ `/evopi-cost detail`（原始 usage/payload/路径）。

**验收判据（Done = 全绿）**：
1. 正常对话一轮后 `/evopi-cost` 显示：命中率、本会话累计 token/成本、上下文压力%+档位、请求数。
2. `.pi/evopi/traces/<traceId>.jsonl` 里能 grep 到 `cost.request` 和 `cost.cache`，字段与决策 4 表一致。
3. 上下文压力人为推高（长对话）跨 80% 时，弹一次 notify 且 JSONL 出现一条 `cost.pressure`（同一档位不重复弹）。
4. `cost.*` **不**出现在 session（`grep evopi.trace` 的 session anchor 里查不到 cost.*）——守 Anchor-only。
5. compaction 后 `getContextUsage()` 为 null 时 `/evopi-cost` 不崩（标「估算不可用」）。

**验收证据（2026-07-01，全绿）**：因本环境跑不起完整 Pi+provider，用「真码离线驱动」自测替代 live run：

- **类型检查**：借 `pi/node_modules` 的 tsc 5.9.3，strict 模式对 `trace.ts`/`cost.ts`/`index.ts` 编译 0 错。
- **纯逻辑单测 19/19**：命中率口径（含除零兜底）、`extractUsage` 只认带 usage 的 assistant entry、压力跨档+同档去重+回落重置+null 兜底。
- **集成测 28/28**：用 fake `pi`/`ctx` 加载**真** `registerCost`，驱动 before/after 钩子后**读磁盘 JSONL** 断言：① 3 事件字段与决策 4 逐字段一致（判据 2）② 压力跨 80% 只弹 1 次 notify + 1 条 `cost.pressure`（判据 3）③ session anchor 里无 `cost.*`（判据 4）④ 摘要含命中率/成本/压力/请求数、null usage 标「估算不可用」不崩（判据 1、5）⑤ 错误响应（无新 assistant）不重复累加、不重复记 `cost.cache`。
- 遗留：真 Pi 会话下的端到端 run 待有环境时补一次（当前逻辑与写盘路径已在真码上验证）。

### 模块 3 — 技能记忆 ✅

**契约**：[03-技能记忆机制.md](../03-技能记忆机制.md)。**冻结**：`memory.*`/`skill.*` 事件；memory 三级 scope；trust 四级；`policy.json` 结构（04 决策 4）。

**拆文件**：`memory.ts`（Memory 层）+ `skill.ts`（Skill 层）+ `policy.ts`（模块 3/4/5 共享，本模块先建，4/5 直接复用）。

**实现清单**：
- [x] `policy.ts`：加载 `.pi/evopi/policy.json`（缺失/坏 JSON 退保守默认；与默认**合并**不替换）+ `scanDangerousText`/`matchDangerousCommand`/`matchProtectedPath` 原语（模块 3 静态扫、4/5 运行时扫共用，决策 9）。
- [x] `memory.ts`：三级 scope（project=`.pi/evopi/memory/`、user=`~/.evopi/memory/`、global V1 占位）；结构化条目解析/序列化（兼容旧 MVP 流水账）；`context` 钩子按关键词+tags+时间邻近本地打分注入 Top-K（CustomMessage、display:false、先滤旧再注入）；`session_before_compact` 抢救成 `candidate`（不注入）；`/evopi-memory` add(`--user/--global/--tag`)/review/摘要。
- [x] `skill.ts`：极简 frontmatter 解析（trust 四级）；`resources_discover` **只返回 trusted 的目录**（additive 语义，candidate/untrusted/blocked 不加载）；危险扫描命中→强制 `blocked`（覆盖声明，决策 9）；`tool_call` read 命中 SKILL.md 路径=一次 invoke（统计采集不用，决策 7）；`/evopi-skill` 列表/review/approve/block。
- [x] index.ts：`registerMemory`/`registerSkill`；删旧 `/evopi-memory` MVP；`memory.write`/`skill.approved` 走 anchor。

**验收判据（Done = 全绿）**：
1. `/evopi-memory add <fact>` 写结构化条目（active、带 scope/tags），JSONL 有 `memory.write` 且**写了 session anchor**；`--user` 写到 `~/.evopi/memory/`。
2. 正常一轮里 `context` 钩子按最后一条 user 文本检索，命中的 active 记忆以 `customType:"evopi-memory"`（display:false）注入 messages，JSONL 有 `memory.retrieve`（**不** anchor）；重复轮不叠加（先滤旧）；无命中不注入。
3. 压缩前 `session_before_compact` 落一条 `status: candidate` 记忆（不注入），`/evopi-memory review` 列得出。
4. `.pi/evopi/skills/<name>/SKILL.md` 中：trusted 被 `resources_discover` 返回加载；candidate/无 trust 字段/命中危险的都**不**加载；JSONL 有 `skill.filtered`（不 anchor）。
5. `/evopi-skill approve <name>` 把 candidate 改 `trust: trusted`（写 `approved_by`），JSONL 有 `skill.approved` 且**写 anchor**；对命中危险的 skill approve 被**拒绝**（强制 blocked）。
6. `skill.*`/`memory.retrieve` 等逐次事件**不**出现在 session anchor（守 Anchor-only，仅 `memory.write`/`skill.approved` anchor）。

**验收证据（2026-07-01，全绿）**：同模块 2，用「真码离线驱动」自测（本环境无 live Pi+provider）：
- **类型检查**：tsc strict + noUnusedLocals/Parameters 对 6 个 .ts（trace/cost/policy/memory/skill/index）编译 **0 错**。
- **集成测 44/44**：fake `pi`/`ctx` 加载**真** registerMemory/registerSkill/policy，读磁盘 MEMORY.md/SKILL.md/JSONL 断言全部 6 条判据 + policy 合并/坏 JSON 兜底 + 危险扫描 + 检索排序/滤旧/无命中 + trust 过滤（含「声明 trusted 但命中危险→强制 blocked」）+ approve 拒危险 + 统计采集（read 命中 SKILL.md）+ 非 skill tool_call 不误计。
- **JSONL 目测**：8 个事件（memory.write/review/retrieve、skill.invoke/outcome/approved/blocked/filtered）字段与决策 10 一致；**anchored 集合恰为 {memory.write, skill.approved}**，Anchor-only 守住。
- **模块 2 回归**：logic 19/19 + 集成 28/28 仍全绿（trace.ts/index.ts 改动无回归）。
- 遗留：真 Pi 会话下 resources_discover/context 端到端 run 待有环境补一次。
